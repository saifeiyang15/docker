# 单人飞行棋游戏使用指南

## 功能概述

单人飞行棋游戏是一个基于掷骰子的回字型棋盘游戏，玩家通过掷骰子在棋盘上移动，触发各种任务获得分数。

## 游戏特点

### 1. 回字型棋盘布局
- 总共 40 个格子，形成回字型布局
- 上边：格子 0-9
- 右边：格子 10-17
- 下边：格子 18-30（从右到左）
- 左边：格子 31-39（从下到上）

### 2. 中心区域
游戏的中心区域包含三个主要部分：
- **Q版角色展示**：显示玩家的可爱角色形象
- **骰子区域**：点击按钮掷骰子，骰子会有动画效果
- **任务文字展示**：显示当前触发的任务信息

### 3. 游戏任务类型
- **前进任务（FORWARD）**：让玩家额外前进若干步
- **后退任务（BACKWARD）**：让玩家后退若干步
- **奖励任务（BONUS）**：获得分数奖励
- **挑战任务（CHALLENGE）**：完成挑战获得高额分数

## 使用方法

### 启动游戏

1. 登录系统后进入游戏大厅
2. 点击"单人游戏"按钮
3. 系统自动创建游戏并初始化

### 游戏操作

1. **掷骰子**
   - 点击中心区域的"掷骰子"按钮
   - 骰子会旋转动画显示随机数字（1-6）
   - 玩家角色自动移动到新位置

2. **触发任务**
   - 当移动到有任务的格子时，会弹出任务详情
   - 任务效果自动执行（前进/后退/加分等）
   - 点击"确定"关闭任务弹窗

3. **查看进度**
   - 顶部显示当前玩家信息、得分和位置
   - 棋盘上当前位置会高亮显示
   - 有任务的格子会显示📋图标

4. **游戏结束**
   - 到达第 40 格时游戏结束
   - 显示最终得分
   - 可以点击"重新开始"按钮再玩一局

### 返回大厅

点击右上角的"返回大厅"按钮可以退出游戏返回大厅。

## 后端 API 接口

### 1. 开始游戏
```
POST /api/game/single/start
请求体：
{
  "userId": 用户ID,
  "username": "用户名"
}
```

### 2. 掷骰子
```
POST /api/game/single/roll
请求体：
{
  "userId": 用户ID
}
```

### 3. 获取游戏状态
```
GET /api/game/single/state/{userId}
```

### 4. 重置游戏
```
POST /api/game/single/reset/{userId}
```

### 5. 初始化任务
```
POST /api/game/single/init-tasks
```

## 自定义配置

### 添加自定义角色图片

1. 将 Q版角色图片放入 `frontend/public/images/` 目录
2. 图片命名格式：`character_xxx.png`
3. 推荐尺寸：100x100 到 200x200 像素
4. 格式：PNG（支持透明背景）

### 修改游戏任务

编辑 `SinglePlayerGameService.java` 中的 `initializeDefaultTasks()` 方法，可以自定义：
- 任务标题和描述
- 任务位置（0-39）
- 任务类型（FORWARD/BACKWARD/BONUS/CHALLENGE）
- 奖励数值

## 技术架构

### 前端
- React + TypeScript
- 响应式设计，支持不同屏幕尺寸
- CSS3 动画效果（骰子旋转、角色移动等）

### 后端
- Spring Boot
- JPA/Hibernate 数据持久化
- RESTful API 设计
- 内存状态管理

## 注意事项

1. 游戏状态存储在服务器内存中，重启服务器会丢失进度
2. 如果角色图片加载失败，会显示默认的绿色笑脸图标
3. 游戏支持多个玩家同时进行，互不干扰
4. 建议在现代浏览器中使用以获得最佳体验
